Detailed Description

Responsible for playback.

Handles playing and enqueueing track sources, advancing to the next track, logging plays with the server, etc.

To observe track changes, position changes, etc., use KVO. For example:

[player addObserver:self forKeyPath:@"currentTrack" options:NSKeyValueObservingOptionNew context:nil];

All of the properties listed above are implemented in a KVO-compliant fashion, so you should be able to observe their changes without any issues.

Continues playing the current track with an option to restart the track if it's already playing.

If the player is already playing, setting shouldRestart to YES will restart the track from the begining.

- (BOOL) updateQueue: (NSArray *)  sourceKeys
withCurrentTrackIndex: (int)  index 

Replace the play queue with a different list of track keys.

This method replaces the entire play queue much like playSources:. Unlike playSources:, this method does not stop playback of the current track.

If the index does not point at the currently playing track, the method will not update the queue and will return NO.

Parameters
sourceKeysList of track keys, such as "t1232"
indexIndex of the currently playing track
Returns
